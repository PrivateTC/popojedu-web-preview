Popojedu — účetní doklady nového webu a napojení na ABRA Gen

Podklad pro účetní · 17. 7. 2026 · Modrý anděl s.r.o. (plátce DPH, DIČ CZ27218422) · připravil Claude (asistent RF), rozhodnutí RF 16.–17. 7. 2026

1) Princip ve dvou větách

Nový web Popojedu (rezervační systém půjčovny) vystavuje zákaznické doklady sám — vlastní číselná řada, vlastní grafika ve firemním stylu, automatické odeslání zákazníkovi. ABRA Gen zůstává jediným účetnictvím — každý doklad z webu se do ní automaticky zapisuje přes její API (rozhraní pro propojení systémů), takže účetní evidence je úplná bez ručního přepisování.

Přílohy dokladů (doplněno 17. 7. na připomínku účetní): K dokladu za přeúčtovanou pokutu se vždy přikládá originální výměr (scan/PDF) — pracovník ho nahraje při zadání pokuty, zákazník ho dostane e-mailem spolu s naším dokladem a do ABRA se doklad zapíše včetně přílohy, takže podklad je přímo u účetního záznamu (kopie i v administraci webu). Stejná možnost příloh platí pro vyúčtování škod (fotky poškození, kalkulace servisu).

Kauce (jistota 10 000 Kč): není výnos a nevzniká k ní daňový doklad. Dvě podoby: (a) blokace na platební kartě — peníze fyzicky nikam nepřijdou, účetně se neděje nic, dokud se případně část nestrhne (pak jde stržená část do vyúčtování); (b) složení převodem — na výpisu KB bude příjem a po vrácení vozu vratka (obvykle týž den po vyúčtování); web ke kauci generuje jen potvrzení o přijetí/vrácení, ne daňový doklad. U segmentu kurýr/přeprava osob se z kauce zadržuje 2 000 Kč po dobu 30 dnů (na pozdější pokuty) — pořád jde o jistotu, ne výnos.

Číselná řada: všechny doklady z webu mají jednu souvislou řadu W2026-#### (rok se mění). Rekapitulace za období vždy doloží úplnost řady (žádné díry).

Důležitá dohoda ABRA tyto doklady přijímá s jejich čísly a nevystavuje k nim vlastní duplicitní doklady. Řady v Abře pro cokoli mimo web (interní faktury, náklady…) zůstávají beze změny.

3) Jak probíhá zápis do ABRA Gen (zvolená varianta)

  • Každý vystavený doklad se ihned zařadí do fronty a systém ho zapíše do ABRA přes její API jako příslušný doklad (faktura vydaná / doklad k přijaté platbě) — se stejným číslem, částkami, DPH rozpisem a vazbou na zákazníka.
  • Zápis je asynchronní: web na Abru nečeká. Když ABRA nebo její API zrovna neběží, zákazník svůj doklad stejně dostane a zápisy se automaticky doženou, jakmile je ABRA dostupná. Nic se neztrácí — u každého dokladu je v administraci webu vidět stav zápisu (✓ zapsáno / čeká).
  • Párování úhrad — automatické, bez ruční práce (rozhodnutí RF 17. 7.): ABRA si denně automaticky importuje elektronický výpis z KB (vestavěný ovladač homebankingu ABRA — bez další licence a bez třetí strany) a při importu sama páruje úhrady s doklady podle variabilního symbolu (VS = číslo rezervace je na dokladu i na platbě). Ručně se řeší jen výjimky (např. platba s chybným VS). Web si nezávisle páruje tytéž pohyby živě (každou hodinu) pro řízení rezervací — dvojí strojová kontrola; denní frekvence v účetnictví plně stačí. K nastavení s vámi: automatické doručování/stahování elektronických výpisů KB ve formátu pro ABRA + zapnutí automatického párování při importu.
  • Volitelný upgrade (jen pokud by import výpisů obtěžoval nebo byla potřeba častější frekvence): přímé napojení ABRA na KB přes bankovní API — služba Finbricks (placená licence „Bankovní API" pro ABRA Gen, cena na vyžádání u obchodu ABRA + smlouva s Finbricks + periodická obnova PSD2 souhlasu), případně služba Dativery (import KB → ABRA à 1 hodina). Doporučujeme nejdřív vyzkoušet denní import a k upgradu sáhnout až podle zkušenosti.
  • Kontrola pro účetní: měsíční rekapitulační sestava (počet dokladů, součty základů a DPH, kontrola souvislosti řady) + kdykoli přístup do administrace webu pro nahlédnutí.
  • Náběhové období: než se API napojení společně odladí (je k dispozici testovací sandbox ABRA), lze doklady přenášet dávkou (export ISDOC/CSV k importu) — čísla i proces jsou stejné, mění se jen způsob přenosu.
Co potřebujeme od vás (účetní):
  1. potvrdit mapování — kódy DPH, předkontace/účty, případně střediska pro doklady z webu;
  2. potvrdit, že instance ABRA Gen má API publikované a je dostupná pro zápis (API klíč máme);
  3. krátký společný test na několika dokladech před ostrým provozem.

4) Zavržená alternativní varianta (pro úplnost)

Varianta B — doklady by vystavovala přímo ABRA: web by při každé platbě zavolal ABRA API, doklad by vznikl v Abře (její číselná řada) a web by zákazníkovi jen přeposlal PDF vytvořené Abrou.

Výhoda: jediný systém vystavuje, žádná dohoda o řadách.

Proč jsme ji zavrhli (rozhodnutí RF 17. 7. 2026):
  1. Závislost webu na nonstop běhu Abry — kdyby ABRA nebo její API nebyly dostupné, zákazník by nedostal doklad a proces rezervace by se zastavil. Půjčovna běží večer i o víkendu; účetní systém takto vytěžovat nechceme.
  2. Vzhled dokladů — PDF z Abry je omezené jejími tiskovými sestavami. Chceme graficky pěkné faktury ve firemním stylu Popojedu (logo, barvy, přehledná struktura), přesně podle našich představ — to umí naše vlastní šablona, a její úpravy dělá náš vývojář, ne konzultant Abry.

Zvolená varianta (bod 3) obě omezení řeší: zákazník dostává hezký doklad z webu okamžitě a nezávisle na Abře, ABRA dostává kompletní účetní zápis automaticky na pozadí.
